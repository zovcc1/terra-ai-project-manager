## 6. Mock/Stub Risk Summary

### 🔴 HIGH Risk (Must fix before production)

| # | Item | Why |
|---|---|---|
| B1 | `EmailService` is a stub | Users **cannot** verify email or reset password. Account registration and password recovery are broken. |
| B3 | Hardcoded AES key in `AiSettingsService` | API keys encrypted with a key baked into source code. Anyone with repo access can decrypt. |

### 🟡 MEDIUM Risk (Should fix for feature completeness)

| # | Item | Why |
|---|---|---|
| B2 | `OpenAiClient` mock mode | Silently returns fake "NONE" responses when API key is missing — no error surfaced to user |
| B4 | `AiInsightsEngine` hardcoded message | LLM is called but its response is discarded; insight text is always the same static Arabic string |
| B7 | `deleteTask` doesn't broadcast WebSocket | After AI deletes a task, other connected users won't see the board update |
| F2–F11 | Dashboard/Project/Admin pages all show hardcoded data | Every management page displays fake data; none connect to real backend endpoints |
| F12 | Demo credentials on login page | Exposes test account passwords in the UI |
| F14–F18 | Task detail, users, teams, members all hardcoded | Admin and member features are non-functional demos |

### 🟢 LOW Risk (Acceptable for now)

| # | Item | Why |
|---|---|---|
| B5 | Cron default mismatch | Overridden by `application.properties`; no functional impact |
| B6 | `DataInitializer` seed data | Standard pattern for dev/test environments |
| F1 | MSW mock handlers | Test infrastructure only; not in production bundle |
| F10–F11 | Chart placeholders | Clearly cosmetic; no functional dependency |
| F13 | Kanban initial tasks | Working local-state demo; drag-and-drop still functions |
